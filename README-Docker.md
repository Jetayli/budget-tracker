# 🐳 Docker Setup for Budget Tracker

This guide will help you set up and run the Budget Tracker application using Docker and Docker Compose.

## 📋 Prerequisites

- [Docker](https://www.docker.com/) (version 20.10 or later)
- [Docker Compose](https://docs.docker.com/compose/) (version 2.0 or later)

## 🚀 Quick Start

### Production Environment

```bash
# Clone the repository (if you haven't already)
git clone <repository-url>
cd Budget-Tracker

# Build and start the application with database
docker-compose up -d

# The application will be available at http://localhost:5000
```

### Development Environment

```bash
# For development with hot reloading
docker-compose -f docker-compose.dev.yml up -d

# The development server will be available at http://localhost:5001
```

## 🔧 Configuration

### Environment Variables

The application uses the following environment variables:

| Variable | Description | Default |
|----------|-------------|---------|
| `NODE_ENV` | Environment mode | `production` / `development` |
| `PORT` | Application port | `5000` / `5001` |
| `DATABASE_URL` | PostgreSQL connection string | Auto-generated by compose |

### Database Configuration

#### Production
- **Database Name**: `budget_tracker`
- **User**: `budget_user`
- **Password**: `budget_password`
- **Port**: `5432`

#### Development
- **Database Name**: `budget_tracker_dev`
- **User**: `dev_user`
- **Password**: `dev_password`
- **Port**: `5433`

## 🗄️ Database Management

### Initial Data

Both environments come with sample data to help you get started:

- **Sample Project**: "Q1 Marketing Campaign" / "Development Marketing Campaign"
- **Categories**: Social Media, Google Ads, Content Marketing, Email Marketing, SEO
- **Subcategories**: Facebook, Instagram, Twitter/X, LinkedIn, Search Ads, etc.
- **Sample Spend Entries**: Pre-populated expense entries

### Database Migrations

The application uses Drizzle ORM for database management. When the containers start:

1. PostgreSQL initializes with the schema from `init.sql` or `init-dev.sql`
2. The application runs `npm run db:push` to ensure schema is up to date
3. Sample data is automatically inserted

## 🛠️ Development Workflow

### Hot Reloading
The development setup supports hot reloading:
- Source code is mounted as a volume
- Changes to your code will automatically restart the development server
- Database data persists across container restarts

## 🔒 Security Considerations

### Production Deployment
- Change default database passwords in `docker-compose.yml`
- Use environment variables for sensitive data
- Consider using Docker secrets for production credentials
- Enable HTTPS in production

### Network Security
- The application runs in a bridge network
- Only necessary ports are exposed
- Database is not accessible from outside the compose network

## 🚀 Deployment Options

### Option 1: Docker Compose (Recommended)
```bash
# Production deployment
docker-compose up -d --build
```

### Option 2: Standalone Docker
```bash
# Build the image
docker build -t budget-tracker .

# Run with external database
docker run -d \
  --name budget-tracker \
  -p 5000:5000 \
  -e DATABASE_URL="postgresql://user:password@host:5432/database" \
  budget-tracker
```

### Option 3: Production with Environment File
```bash
# Create .env file
echo "DATABASE_URL=postgresql://user:password@host:5432/database" > .env

# Run with environment file
docker-compose --env-file .env up -d
```

## 🐛 Troubleshooting

### Common Issues

#### Port Conflicts
```bash
# Error: Port already in use
# Solution: Change ports in docker-compose.yml
ports:
  - "5002:5000"  # Use different host port
```

#### Database Connection Issues
```bash
# Check database container status
docker-compose ps postgres

# Restart database
docker-compose restart postgres

# View database logs
docker-compose logs postgres
```

#### Build Issues
```bash
# Clear Docker cache
docker system prune -a

# Rebuild without cache
docker-compose build --no-cache
```

#### Permission Issues
```bash
# Fix file permissions (Linux/Mac)
sudo chown -R $USER:$USER .

# Or use Docker with proper user mapping
docker-compose run --rm --user $(id -u):$(id -g) app whoami
```

### Performance Optimization

#### Production Optimizations
```bash
# Use multi-stage builds for smaller images
# Already implemented in Dockerfile

# Enable resource limits
docker-compose up -d --scale app=2 --memory=1g
```

## 📝 Customization

### Adding Custom Environment Variables
```yaml
# docker-compose.yml
services:
  app:
    environment:
      - CUSTOM_VAR=value
      - ANOTHER_VAR=${EXTERNAL_VALUE}
```

### Custom Database Initialization
```bash
# Replace init.sql with your custom schema
# Mount your SQL file
volumes:
  - ./custom-init.sql:/docker-entrypoint-initdb.d/init.sql
```

## 🔄 Backup and Restore

### Database Backup
```bash
# Backup production database
docker exec budget-tracker-db pg_dump -U budget_user budget_tracker > backup.sql

# Backup development database
docker exec budget-tracker-dev-db pg_dump -U dev_user budget_tracker_dev > dev-backup.sql
```

### Database Restore
```bash
# Restore to production database
docker exec -i budget-tracker-db psql -U budget_user budget_tracker < backup.sql

# Restore to development database
docker exec -i budget-tracker-dev-db psql -U dev_user budget_tracker_dev < dev-backup.sql
```
